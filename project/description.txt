Добрый день!
Представляю свою работу - база данных для онлайн сервиса по продаже автобусных билетов.

БД содержит 10 таблиц:
1. analytics - данные с источников привлечения трафика на сайт, поступают из Google Analytics
2. bookings - данные о бронированиях клиентов, содержит информацию о статусе бронирования, мест 
	отправления и прибытия, стоимости заказа, вермя отправления, данные о клиенте
3. carriers - список перевозчиков, осуществляющие рейсы
4. cities - список городов
5. customers - основные данные о клиентах
6. device_type - список типов устройств пользователей
7. orders - данные о заказах. Содержит transaction_id для связки с данными из Google Analytics, 
	id покупателя, информацию о применённом промокоде
8. profiles - расшренные данные о пользователе
9. routes - список рейсов с id городов и стоимостью билета
10. transactions - информация собираемая в backend сайта. Вспомогательная таблица для совмещения orders и analytics

Дамп БД содержит уже обработанные данные. Скрипт с командами обработки также прикладываю.
Фантазии на разнообразные выборки не хватило, привёл примеры наиболее актуальных. Описание запросов в файле с
самими запросами.
